Term 2

Evaluation of the Final Solution

We are quite happy with the outcome, as we managed to hit all the key requirements. We made sure that at every step of the way, the user knows exactly what they are doing, and what information they are looking at. Our data visualisation is straightforward and is based on charts that are familiar to the average person. There is no need to learn how to read the data analysis, which makes the system easy to use for both experienced and inexperienced users.

We even managed to add in some extra functionality. The pie-chart is interactive, so users can get much more specific information. When clicking on the green slice, you get a chart that accounts only for the risk-free documents Also, both charts can be exported. This can help with end-of-the-project reports or with justifying different decisions and keeping track of the evidence backing them up.

We really think that once put together, this project can have a great impact on the industry, by automating a very tedious and error-prone process.

Iterations and Forks in Prototypes

FIRST ITERATION

The first iteration of the app was the tabbed menu as well as the Home Page that are colour coded with simple text display. We used the main colours from Clifford Chance website. This proved to be very useful to show to our client how we envisioned the app and discuss on what should be included next.

SECOND ITERATION

The second iteration of the app had more details in every tab of the web app especially home page, clusters overview, document overview and saved document. This enables us to show our client the way the app should be working and obtain feedbacks from him so that we could improve.

THIRD ITERATION

The third iteration of the app involves the change of homepage display to show the list of projects in a more elegant and efficient way. We changed the horizontal tabs to vertical tabs to avoid the problem with display space.

FOURTH ITERATION

During this iteration, we created login, register and forgot password pages, however, having discussed it with our client, we decided to take this out because it is not vital for the web app.

FIFTH ITERATION

The fifth iteration of the app involves implementation of cluster pie chart and 2D table to display clusters and documents. This is one of the key challenge of the app as the we were unable to obtain accurate and reliable results from team 39 and team 40 for visualisation. After having discussion with our client, we decided to use mock data.

SIXTH ITERATION

Our sixth iteration was one of the longest ones, as we were unable implement a database that make perfect sense to the app. Finally, we decided to use Django as the backend and AngularJS as the front-end.

SEVENTH ITERATION

The seventh iteration fixes several linking issues for the visualisation and the database.

Term 1

Our initial idea is to create an app that is easy to use on all desktop platforms. To find out the best solution, we researched on Cross-Platform Desktop Application to find out a viable solution:

Java

Java originates at Sun Microsystems, in 1991. James Gosling was trying to find a way of developing C++ programs that could run on the widest possible variety of platforms without recompiling. The original Java mantra was "write once, run anywhere". It no doubt offers one of the best cross platform versatility. It is also one of the most commonly used programming languages in the world.

Advantages:

+ only need to be written once

+ no extra compiling time for every Operating System

Disadvantages:

- user machine must have Java Virtual Machine (JVM) installed

Web Application

Web applications run on a server so it does not require the user to install anything. It therefore offers the highest portability as long as the right version of code is deployed on the server.

Django

Django is a high level web framework based on Python that encourages rapid development and clean, pragmatic design. Built by experienced developers, it takes care of much of the hassle of Web development with a lot of built in functions that make life easier. It also comes with SQLite, which is free and easy to use.

Microsoft Azure

Windows Azure is just not an OS for Cloud but can be used for Data centers, application software which will integrate the resources in the best efficient way. Microsoft offers a feature that allows the applications to be stuck on top of the Azure platform and perform effortlessly. It also comes with an SQL database.

Conclusions:

After understanding the subject matter, we decided to go for a Web Application because of its versatility on any desktop OS. Not to mention it will be such a hassle for the whole company (Clifford Chance) to install JVM. Before we get the developer license from our client for Microsoft Azure, we will go ahead and use Django as the back-end as we have prior experience dealing with Django and it is user friendly.

Since creating a Web Application is our final choice, we continued to research and learn:

Version Control

We decided to use Git for our version control system. We will be using GitHub to host all our source code. We will have 4 branches, namely Master, Backup, Dev1 and Dev2. While Master will have the latest working version of the app, Backup will always be storing the last working version (The last milestone reached). Dev1 and Dev2 are for development purposes so that both of us can work on the app at the same time. After we finished, results from Dev1 and Dev2 will be merged with selection into Master. Automated updates will also be done from Github to the server in future.
